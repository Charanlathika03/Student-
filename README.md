# SRS (Procedure)
# 1. Introduction

A Student Enrollment System manages the admission process of students in an educational institution.
It handles registration, course selection, fee payment, document verification, and record maintenance.

# 2. Purpose

The purpose of the system is to:

Automate student admission

Maintain student academic and personal details

Provide efficient course enrollment

Reduce manual errors during data handling

Improve accessibility for staff and administration

# 3. Scope

This system covers:

Student registration

Course enrollment and subject selection

Document submission and verification

Fee payment and receipt generation

Student ID creation

Admin and staff management

Reports and academic records

# 4. Functional Requirements

Students must register with personal and academic details.

System validates documents and eligibility.

Students can select available courses and subjects.

Payment module must generate fee receipts.

Admin manages courses, departments, staff, and student data.

Staff can assist students with enrollment.

# 5. Non-Functional Requirements

Security: Student data must be protected.

Usability: Interface should be easy for students and staff.

Performance: Fast data processing and retrieval.

Scalability: System must support large student counts.

Accuracy: Prevents duplicate or invalid registrations.

# 6. Assumptions

Students provide correct and valid information.

Courses are predefined by the institution.

System has stable internet/database connectivity.

Admin has full access rights, staff has limited access.
